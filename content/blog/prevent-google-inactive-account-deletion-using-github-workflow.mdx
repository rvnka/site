## Introduction

Since December 2023, Google has enforced a policy of permanently deleting accounts that have been inactive for two full years. While this makes sense from a security and resource standpoint, it poses a real risk for anyone with secondary accounts, legacy credentials, or accounts tied to important services they rarely sign into.

This guide walks through an automated solution using GitHub Actions that keeps your Google accounts active by sending a self-addressed email every month. It supports multiple accounts simultaneously, handles failures gracefully with retry logic, and runs entirely free on GitHub's infrastructure.

---

## Why Accounts Get Deleted

Google considers an account inactive if the owner hasn't performed any of the following activities within two years: reading or sending email in Gmail, using Google Drive or Workspace apps, watching YouTube while signed in, downloading apps from the Play Store, searching while logged into Google, or signing in to third-party services via Google.

Once deleted, an account cannot be recovered. All associated data including emails, Drive files, and photos is gone permanently.

---

## How the Solution Works

The approach is straightforward:

1. Create a private GitHub repository to host the workflow.
2. Store account credentials securely in GitHub Secrets.
3. Run an automated monthly workflow that sends an email from each account using Gmail's SMTP server.

Google registers this as legitimate account activity, keeping each account well within the two-year activity window.

---

## Step 1: Create a GitHub Repository

Go to [github.com](https://github.com), click **New**, and create a repository with any name you like (e.g. `gmail-ping`). Set the visibility to **Private**, then click **Create repository**.

Keeping the repository private is a good security habit even though credentials are stored in Secrets rather than in code.

---

## Step 2: Generate App Passwords

This workflow uses App Passwords rather than your primary Google account password. An App Password is a 16-character code that Google generates for third-party applications. It can be revoked at any time without changing your main password and does not bypass two-factor authentication.

**Before you begin:** Two-Step Verification must be enabled on your Google account.

1. Open [myaccount.google.com](https://myaccount.google.com) and navigate to **Security**.
2. Use the search bar on the page and search for **App Passwords**.
3. Give the app a name (e.g. `GitHub Workflow`) and click **Create**.
4. Copy the 16-character code shown. It will only be displayed once, so save it somewhere safe.

Repeat this process for every Google account you want to protect.

---

## Step 3: Configure the `MAIL_CONF` Secret

All account credentials are stored in a single GitHub Secret named `MAIL_CONF`, using a simple key-value format. This lets you register multiple accounts in one place.

```
PING_TO=target@gmail.com

GMAIL_USER_1=account1@gmail.com
GMAIL_PASS_1=abcd efgh ijkl mnop

GMAIL_USER_2=account2@gmail.com
GMAIL_PASS_2=qwer tyui opas dfgh

# Comment lines are ignored
# Add more accounts using _3, _4, and so on
```

`PING_TO` is the recipient address for all outgoing pings. You can set it to one of your own accounts. `GMAIL_USER_N` and `GMAIL_PASS_N` are the email address and App Password for the Nth account, with indexes starting at 1.

To save this secret, open your repository on GitHub, go to **Settings -> Secrets and variables -> Actions**, click **New repository secret**, name it `MAIL_CONF`, paste the configuration, and save.

## Step 4: Create the Workflow File

Inside your repository, create `.github/workflows/ping.yml` with the following content:

```yaml
name: "Ping!"

on:
  schedule:
    - cron: "0 0 1 * *"
  workflow_dispatch:

jobs:
  ping:
    runs-on: ubuntu-latest

    steps:
      - name: Secure Gmail Ping
        shell: bash
        env:
          CONF: ${{ secrets.MAIL_CONF }}
          MAX_WORKERS: "3"
          RATE_INTERVAL: "4"
          MAX_RETRY: "3"
          BASE_DELAY: "5"
        run: |
          set -euo pipefail
          set +x

          python3 - <<'EOF'
          import os
          import sys
          import smtplib
          import ssl
          import time
          import random
          import traceback
          from datetime import datetime, timezone
          from email.message import EmailMessage
          from email.utils import formatdate, make_msgid
          from concurrent.futures import ThreadPoolExecutor, as_completed
          from threading import Lock

          SMTP_SERVER = "smtp.gmail.com"
          SMTP_PORT = 587
          TIMEOUT = 20

          MAX_WORKERS = int(os.environ.get("MAX_WORKERS", "3"))
          RATE_INTERVAL = float(os.environ.get("RATE_INTERVAL", "4"))
          MAX_RETRY = int(os.environ.get("MAX_RETRY", "3"))
          BASE_DELAY = float(os.environ.get("BASE_DELAY", "5"))

          last_send_time = 0.0
          rate_lock = Lock()

          def log(*args):
              print(datetime.now(timezone.utc).isoformat(), "-", *args, file=sys.stderr)

          def parse_config(conf_text):
              config = {}
              for raw in conf_text.splitlines():
                  line = raw.strip()
                  if not line or line.startswith("#"):
                      continue
                  if "=" not in line:
                      log("Skipping malformed config line:", line)
                      continue
                  key, value = line.split("=", 1)
                  config[key.strip()] = value.strip().strip('"').strip("'")
              return config

          def extract_accounts(config):
              accounts = []
              index = 1
              while True:
                  user = config.get(f"GMAIL_USER_{index}")
                  passwd = config.get(f"GMAIL_PASS_{index}")

                  if not user and not passwd:
                      break

                  if user and passwd:
                      accounts.append((user, passwd))
                  else:
                      log(f"Incomplete account index {index}, skipping.")

                  index += 1
              return accounts

          def build_message(sender, recipient, idx):
              now = datetime.now(timezone.utc)

              msg = EmailMessage()
              msg["From"] = sender
              msg["To"] = recipient
              msg["Subject"] = f"Ping! ({idx})"
              msg["Date"] = formatdate(localtime=True)
              msg["Message-ID"] = make_msgid()
              msg["X-Mailer"] = "Automated Mail Service"
              msg.set_content(
                  f"Hello, {recipient}\n\n"
                  "This is an automated system ping.\n\n"
                  f"Timestamp (UTC): {now.isoformat()}\n\n"
                  "Best regards,\n"
                  f"{sender} ({idx})\n"
              )
              return msg

          def wait_for_rate_slot():
              global last_send_time
              with rate_lock:
                  now = time.monotonic()
                  diff = now - last_send_time
                  if diff < RATE_INTERVAL:
                      sleep_time = (RATE_INTERVAL - diff) + random.uniform(0, 0.5)
                      time.sleep(sleep_time)
                  last_send_time = time.monotonic()

          def send_email(user, password, recipient, idx):
              wait_for_rate_slot()
              msg = build_message(user, recipient, idx)

              context = ssl.create_default_context()
              with smtplib.SMTP(SMTP_SERVER, SMTP_PORT, timeout=TIMEOUT) as server:
                  server.ehlo()
                  server.starttls(context=context)
                  server.ehlo()
                  server.login(user, password)
                  server.send_message(msg)

          def send_with_retry(user, password, recipient, idx):
              for attempt in range(1, MAX_RETRY + 1):
                  try:
                      send_email(user, password, recipient, idx)
                      log(f"[{idx}] SUCCESS:", user)
                      return True
                  except Exception as e:
                      err = "".join(traceback.format_exception_only(type(e), e)).strip()
                      log(f"[{idx}] Attempt {attempt} failed for {user}: {err}")
                      if attempt < MAX_RETRY:
                          delay = BASE_DELAY * (2 ** (attempt - 1)) + random.uniform(0, 1)
                          time.sleep(delay)
                      else:
                          log(f"[{idx}] FINAL FAILURE:", user)
                          return False

          def main():
              conf = os.environ.get("CONF")
              if not conf:
                  log("MAIL_CONF missing (critical)")
                  sys.exit(1)

              config = parse_config(conf)
              ping_to = config.get("PING_TO")

              if not ping_to:
                  log("PING_TO missing (critical)")
                  sys.exit(1)

              accounts = extract_accounts(config)
              if not accounts:
                  log("No valid accounts found (graceful exit)")
                  sys.exit(0)

              failed = 0

              with ThreadPoolExecutor(max_workers=MAX_WORKERS) as executor:
                  futures = []
                  for idx, (user, passwd) in enumerate(accounts, start=1):
                      futures.append(
                          executor.submit(send_with_retry, user, passwd, ping_to, idx)
                      )

                  for future in as_completed(futures):
                      try:
                          if not future.result():
                              failed += 1
                      except Exception as e:
                          log("Unhandled worker exception:", e)
                          failed += 1

              if failed:
                  log(f"{failed} account(s) failed. Workflow completed (fail-safe).")
              else:
                  log("All accounts processed successfully.")

              sys.exit(0)

          if __name__ == "__main__":
              main()
          EOF
```

## How It Works

**Scheduling.** The `cron: "0 0 1 * *"` trigger runs the workflow at midnight UTC on the first of every month. This is frequent enough to keep all accounts well within the two-year inactivity limit. The `workflow_dispatch` option lets you trigger a run manually from the Actions tab whenever needed.

**Configuration parsing.** The `parse_config()` function reads `MAIL_CONF` line by line, skipping blank lines and comments. `extract_accounts()` then pulls out credential pairs by scanning for `GMAIL_USER_N` and `GMAIL_PASS_N` keys sequentially, starting at index 1.

**Encrypted delivery.** Every email is sent over `smtp.gmail.com` port 587 using STARTTLS, the standard encryption protocol for SMTP. Python's default SSL context validates Gmail's certificate automatically, so credentials and message content are protected in transit.

**Rate limiting.** `wait_for_rate_slot()` uses a thread lock to ensure at least `RATE_INTERVAL` seconds pass between sends, even when multiple accounts are processed in parallel. This prevents Gmail from temporarily blocking connections due to bursts of activity.

**Retry with exponential backoff.** Failed sends are retried up to `MAX_RETRY` times. The delay between attempts follows the formula `BASE_DELAY x 2^(attempt - 1) + random(0, 1)`. With the default `BASE_DELAY` of 5, retries happen at roughly 5, 10, and 20 seconds. The random jitter prevents multiple accounts from retrying at exactly the same time.

**Environment variables.** You can tune the workflow's behavior by adjusting the env block: `MAX_WORKERS` controls how many accounts run in parallel (default 3), `RATE_INTERVAL` sets the minimum gap between sends in seconds (default 4), `MAX_RETRY` sets the maximum retry attempts per account (default 3), and `BASE_DELAY` is the base delay in seconds for the backoff calculation (default 5).

**Fail-safe exit.** Even if one or more accounts fail after exhausting all retries, the script exits with code 0. This is intentional: it prevents GitHub from marking the entire run as failed just because a single account had a transient issue.

## Troubleshooting

**"Username and Password not accepted"** means you are using your regular Google password instead of an App Password, or the App Password was copied incorrectly. Double-check that Two-Step Verification is active and regenerate the App Password if needed.

**"MAIL_CONF missing (critical)"** means the secret has not been saved, or the name doesn't match exactly. The name must be `MAIL_CONF` in all capitals.

**"Incomplete account index N, skipping"** means one of your `GMAIL_USER_N` or `GMAIL_PASS_N` fields is missing from the configuration. Both must be present for every index, starting from 1.

**Scheduled runs stop working** after 60 days of repository inactivity (no commits). GitHub sends an email notification before disabling the schedule. To re-enable it, open the Actions tab and turn the workflow back on. **(I will fix this later!)**

## Security Notes

Never commit App Passwords directly into code. Always use GitHub Secrets. It is also a good practice to use a different App Password for each workflow so individual ones can be revoked without affecting others. Review your active App Passwords in your Google Account settings periodically and remove any that are no longer needed.

## Conclusion

This workflow gives you a hands-off way to keep multiple Google accounts active. It runs on a set schedule, encrypts all traffic in transit, retries failures intelligently, and never exposes credentials in logs. As long as the repository is active and the secret is configured correctly, your accounts will stay well within Google's activity requirements.
